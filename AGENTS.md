# Repository Instructions

## Commits

- Use Angular-style commit messages for all commits.
- Format commit subjects as `type(scope): summary` when a scope is useful, otherwise `type: summary`.
- Keep the summary imperative and concise.
- Preferred types include `feat`, `fix`, `docs`, `refactor`, `test`, `build`, `ci`, and `chore`.
- Allowed unconventional type 'wip'

Examples:

- `fix(config): add explicit Node process import`
- `feat(upload): support attached data files in chat`
